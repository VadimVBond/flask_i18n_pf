# PROJECT_CONTEXT.md
Архитектура и контекст проекта

## Назначение проекта

Этот проект представляет собой многоязычный сайт-портфолио, построенный на Flask с использованием Markdown-контента.

Основные цели проекта:

- отображение Markdown-страниц
- поддержка нескольких языков
- возможность генерации статической версии сайта
- простая структура контента
- возможность масштабирования (портфолио, блог, документация)

---

## Основные технологии

- Flask
- Flask-FlatPages
- Flask-Frozen
- Jinja2 templates
- Markdown

---

## Поддерживаемые языки

Сайт поддерживает следующие языки:

- en
- ru
- uk

Список языков задаётся в коде:

```

SUPPORTED_LANGS = ["en","ru","uk"]

```

---

## Структура проекта

```

project/

mysite.py
config.py

content/
posts/
en/
ru/
uk/

templates/
base.html
index.html
post.html

static/
css/
js/
img/

```

---

## Контент

Контент хранится в Markdown-файлах.

Пример структуры:

```

content/posts/en/my-post.md

```

Файл должен содержать метаданные:

```

title: Post title
date: YYYY-MM-DD
lang: en
slug: post-name

```

---

## URL структура

Сайт использует языковые префиксы:

```

/en/
/ru/
/uk/

```

Примеры страниц:

```

/en/posts/
/ru/posts/
/uk/posts/

```

Отдельная статья:

```

/en/posts/my-post/

```

---

## Основной поток работы

1. Markdown читается через Flask-FlatPages
2. Контент передаётся в шаблоны Jinja2
3. Flask рендерит страницы
4. При необходимости сайт может быть заморожен Flask-Frozen

---

## Генерация статического сайта

Для генерации используется:

```

flask freeze

```

Статический сайт появляется в папке:

```

build/

```

---

## Важные ограничения

- структура папок не должна изменяться
- язык страницы должен совпадать с папкой
- slug должен быть уникальным
- шаблоны должны наследоваться от base.html
