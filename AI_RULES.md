# AI_RULES.md
Правила редактирования проекта для AI-ассистентов

## Общие правила

AI должен:

- изменять только необходимые файлы
- избегать дублирования кода
- не ломать существующую архитектуру
- не изменять структуру проекта без запроса

---

## Ограничения

AI не должен:

- удалять Markdown-контент
- изменять slug страниц
- менять язык страницы
- создавать дубли маршрутов Flask
- добавлять повторяющиеся строки кода
- добавлять лишние импорты

---

## Работа с Python кодом

Следовать правилам:

- PEP8
- минимальная сложность функций
- понятные имена переменных

Пример:

Плохо:

```

x = pages

```

Хорошо:

```

posts = pages

```

---

## Работа с шаблонами

Использовать:

- Jinja2
- наследование шаблонов
- минимум логики

Структура:

```

base.html
└ index.html
└ post.html

```

---

## Работа с маршрутами

Маршруты не должны дублироваться.

Пример корректного маршрута:

```

@app.route("/<lang>/posts/<slug>/")

```

---

## Проверка после изменений

После изменений проект должен запускаться:

```

python mysite.py

```

Проверить:

- сервер запускается
- страницы открываются
- Markdown отображается
- переключение языка работает
