# AGENTS.md
Руководство для AI-ассистентов, работающих с этим репозиторием.

## Обзор проекта

Этот проект представляет собой многоязычный сайт, построенный с использованием:

- Flask
- Flask-FlatPages (Markdown-контент)
- Flask-Frozen (генерация статического сайта)
- шаблоны Jinja2
- поддержка i18n (несколько языков)

Основная цель:
Отображение Markdown-контента в виде веб-страниц и возможность генерации статической версии сайта.

Поддерживаемые языки:

- en
- ru
- uk


## Структура проекта

mysite.py  
Основное приложение Flask.

content/
Markdown-страницы.

Пример структуры:

content/posts/en/post1.md  
content/posts/ru/post1.md  
content/posts/uk/post1.md  

templates/
Jinja-шаблоны.

static/
CSS, JS, изображения.

config.py (опционально)
Конфигурация проекта.


## Правила для Markdown-контента

Markdown-файлы должны содержать метаданные:

```

title: Заголовок
date: YYYY-MM-DD
lang: en
slug: имя-страницы

```

Пример:

```

title: Привет
date: 2026-01-01
lang: ru
slug: privet

```

Язык в метаданных должен соответствовать языковой папке.


## Правила маршрутов

Маршруты определяются в файле `mysite.py`.

Типичная структура URL:

```

/<lang>/
/<lang>/posts/
/<lang>/posts/<slug>/

```

Язык должен проверяться по списку:

```

SUPPORTED_LANGS = ["en", "ru", "uk"]

```

Если язык не поддерживается → выполняется редирект на язык по умолчанию.


## Правила редактирования для AI

AI-ассистенты должны соблюдать следующие правила:

1. Не дублировать существующий код.
2. Не изменять работающие маршруты без явного запроса.
3. Не менять структуру папок.
4. Не переименовывать шаблоны без необходимости.
5. Не добавлять повторяющиеся импорты.
6. Изменять только те файлы, которые относятся к задаче.


## Стиль кода

Python:

- соблюдать PEP8
- использовать простые функции
- избегать лишних абстракций

Шаблоны:

- использовать Jinja2
- минимизировать логику в шаблонах
- предпочитать циклы вместо повторяющихся блоков


## Типичные задачи

AI-ассистенты могут выполнять:

- добавление новых маршрутов
- добавление языковой поддержки
- улучшение рендеринга Markdown
- добавление пагинации
- добавление страниц портфолио
- исправление ошибок шаблонов

Ассистенты должны изменять **только необходимые участки кода**.


## Тестирование

После изменений проект должен запускаться командой:

```

python mysite.py

```

Ожидаемое поведение:

- сервер запускается
- маршруты работают
- Markdown-страницы корректно отображаются
- переключение языков работает


## Генерация статического сайта

Если используется статическая сборка:

```

flask freeze

```

Сгенерированный сайт должен появиться в папке:

```

build/

```


## Правила безопасности

AI-ассистенты не должны:

- удалять Markdown-контент
- менять языковые slug
- ломать наследование шаблонов
- создавать дубли маршрутов Flask
- добавлять повторяющиеся строки кода

Если возникает неопределённость — ассистент должен запросить уточнение.
