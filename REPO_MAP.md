```md
# REPO_MAP.md
Карта репозитория

Этот файл описывает структуру проекта и указывает AI-ассистентам,
в какие файлы можно вносить изменения.

---

# Главный файл Flask-приложения в разных проектах может называться:

app.py
mysite.py
main.py
server.py 

Содержит:

- создание Flask app
- маршруты (routes)
- загрузку Markdown
- настройку языков

AI может изменять:

- функции маршрутов
- обработку контента
- логику выбора языка

AI не должен:

- удалять основной app
- дублировать app.run()

---

config.py  
Конфигурация проекта.

Может содержать:

- список языков
- настройки Markdown
- настройки Flask

AI может добавлять новые параметры.

---

# Контент

content/

Папка содержит Markdown-контент.

Пример:

content/posts/en/post1.md
content/posts/ru/post1.md
content/posts/uk/post1.md

AI не должен:

- удалять контент
- менять язык файла
- менять slug страниц

AI может:

- читать структуру
- использовать метаданные страниц

---

# Шаблоны

templates/

Содержит Jinja-шаблоны.

Основные файлы:

base.html  
Базовый шаблон.

index.html  
Главная страница.

post.html  
Страница статьи.

AI может:

- исправлять HTML
- улучшать Jinja код

AI не должен:

- ломать наследование base.html

---

# Статические файлы

static/

Содержит:

css/
js/
img/

AI может:

- добавлять CSS
- добавлять JS

AI не должен:

- удалять существующие файлы.

---

# Генерация статического сайта

build/

Эта папка создаётся автоматически.

AI не должен изменять её содержимое.

---

# Основные маршруты сайта

Главная:

/<lang>/

Список постов:

/<lang>/posts/

Страница поста:

/<lang>/posts/<slug>/

---

# Поддерживаемые языки

Список языков:

```

SUPPORTED_LANGS = ["en","ru","uk"]

```

AI должен проверять язык перед использованием.

Если язык неверный → redirect.

---

# Правила изменений

AI должен:

- изменять только необходимые файлы
- избегать дублирования строк
- не создавать повторяющиеся маршруты
- сохранять структуру проекта

---

# Проверка после изменений

После изменения кода необходимо проверить:

```

python mysite.py

```

Ожидаемый результат:

- сервер запускается
- страницы открываются
- Markdown отображается
- язык переключается
```
